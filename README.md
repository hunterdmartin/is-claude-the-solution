# Is Claude the Solution?

An experiment in whether a large language model, given a real and currently
active global crisis, can produce a genuinely useful path toward solving it —
and whether changing the *cognitive style* of the model changes the quality,
creativity, or viability of the solutions it generates.

## The crisis

Both arms of this experiment are tasked with the **same** problem, fixed in
advance so that neither arm has the advantage of a different (easier or harder)
target:

> **Antimicrobial resistance (AMR)** — the global, accelerating loss of
> effectiveness of antibiotics, antivirals, antifungals, and antiparasitics as
> the pathogens they target evolve resistance. AMR is already associated with
> well over a million deaths per year, threatens to unwind a century of modern
> medicine (surgery, chemotherapy, childbirth, transplantation all depend on
> working antibiotics), and is worsening faster than the drug pipeline can
> replace.

### No constraints on the solution

This is the defining rule of the experiment. **Neither instance is restricted
in any way in the solution it may propose**, beyond the model's own internal
safeguards and values. There is no required format, no banned class of idea, no
"realistic only" filter, no scope limit. Scientific, political, economic,
social, biological, computational, infrastructural, speculative — all are fair
game. Each instance has full creative reign to dream up whatever solution is
within its power to conceive. The point is to see what each one actually does
with total freedom.

## The two arms

The experiment runs as two isolated branches of this repository. Each is worked
by an independent Claude Code (Opus 4.8) instance that never sees the other.

| Branch | Instance | Configuration |
|---|---|---|
| `status-claude` | Baseline | An unmodified Claude Code instance — the control / status quo. |
| `intoxiclaude` | Variant | A Claude Code instance running in a modified configuration. |

Both start from the identical `main` scaffolding (this README and the shared
`CLAUDE.md`), receive the identical crisis and the identical no-constraints
mandate, and then diverge only by their configuration.

## Isolation (why the branches can't see each other)

For the comparison to mean anything, the two arms must be **blind to each
other**. No cross-pollination of ideas, no anchoring on what the other arm
produced. Each instance:

- works **only** on its own branch,
- must **not** inspect, check out, diff, fetch, or read any other branch, and
- must **not** read any solution content generated by the other arm.

These rules are enforced by convention in `CLAUDE.md`. See that file for the
binding instructions each instance must follow.

## Evaluation

After both arms have run, their outputs are compared by a human observer:
breadth of ideas, originality, internal coherence, and plausibility of impact
against AMR — with particular attention to whether the configuration difference
produced a meaningful difference in the work.
